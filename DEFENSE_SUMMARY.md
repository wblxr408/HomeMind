# HomeMind 答辩摘要

## 1. 一句话定位

HomeMind 不是一个只能执行固定命令的智能家居控制器，而是一个**端侧优先、可选端云协同、可持续学习、可渐进自动化**的家庭智能体。

## 2. 核心亮点

### 端侧优先，但不是纯规则系统

系统保留了自然语言主链路：

- `BSR` 候选召回
- `LSR` 轻量精排
- `LLM` 结构化决策

它能理解“有点闷”“像昨天晚上那样”这类家庭场景下的模糊表达，而不是只支持固定模板命令。

### 云端可协同，但端侧掌握执行权

我们新增了：

- `InferenceRouter`
- `PrivacyRedactor`
- `CommandValidator`

系统会先在端侧完成候选压缩和上下文整理，再决定是否上云；即使云端参与，也只负责裁决建议，最终执行仍必须经过端侧校验。

### 记忆不是简单日志堆积，而是分层持久化

我们把记忆拆成三层：

- `SessionStore`：短期会话状态
- `PreferenceStore`：长期结构化偏好
- `KnowledgeBase`：可检索文本记忆

这样系统既能记住“最近发生了什么”，也能逐渐形成“用户长期偏好是什么”。

### 语音能力适合低资源端侧

语音输入采用：

- 浏览器录音
- 服务端本地 `Vosk` 转写
- `LanguageNormalizer` 做中英/口语/方言归一化

相比直接依赖浏览器语音 API，这种方式更可控；相比更重的 ASR 模型，也更适合低内存设备。

### 自动化能力是渐进引入，而不是推翻 AI 主链路

我们新增了：

- `TAPRuleStore`
- `TAPEngine`
- 规则自动调度器

这样系统同时拥有两条能力：

- 自然语言理解链路，处理模糊用户表达
- TAP 自动化链路，处理定时和环境触发

两者复用同一套执行层和命令校验层。

## 3. 当前系统链路

```text
用户输入 / 语音输入
-> 归一化
-> BSR
-> LSR
-> InferenceRouter
-> local / cloud / clarify / fallback
-> PrivacyRedactor（cloud path）
-> LLMDecider / CloudClient
-> CommandValidator
-> 执行层
-> SessionStore / PreferenceStore / KnowledgeBase 写回
```

自动化链路：

```text
TAPRuleStore
-> TAPEngine
-> Scheduler
-> CommandValidator
-> 执行层
```

## 4. 当前已经实现的答辩点

- 模糊意图理解
- 端云协同路由
- 执行前安全校验
- 结构化上下文持久化
- 长期偏好学习
- 本地语音识别
- 方言/口语归一化基础版
- 自动化规则管理与自动调度
- Web 可视化界面

## 5. 可以这样总结

> HomeMind 从原型式智能家居演示系统，升级为一个端侧优先、可选端云协同、具备持续学习能力和渐进式自动化能力的家庭智能体。端侧完成候选召回、路由决策、隐私脱敏、命令校验和执行控制，云侧只负责复杂模糊意图裁决；同时系统通过结构化记忆、语音反馈、偏好学习和 TAP 自动化，逐步形成更贴近真实家庭场景的连续体验。
